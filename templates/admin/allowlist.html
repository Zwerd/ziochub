Allowlist (Known‑Good / Critical Assets)

Indicators in this allowlist are blocked from submission across all import methods (manual, bulk, JIRA, CSV, API). Existing IOCs that match will appear as anomalies in Feed Pulse. Use this to prevent accidental submissions like google.com.

Supported formats

  • google.com — suffix match (google.com + all subdomains)
  • suffix:google.com — explicit suffix match
  • domain:example.com — exact domain only (no subdomains)
  • tld:gov — all domains under .gov TLD
  • 8.8.8.8 — exact IP address
  • ip:8.8.8.8 — explicit exact IP
  • 10.0.0.0/8 — CIDR range (all IPs in subnet)
  • cidr:10.0.0.0/8 — explicit CIDR range
  • Lines starting with # are comments.

Behavior

  • Submission: Any IOC matching this list is blocked — no exceptions.
  • Feed Pulse: Existing Domain/URL IOCs matching this list appear as anomalies.
  • Anomalies: Separate from allowlist — private IPs, short domains, etc. are always flagged independently.
  • After saving, the server cache is refreshed automatically.
